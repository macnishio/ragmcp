# 【重要】リリース手順マニュアル

## リリースはGitHub Actions CIで全自動

ragmcpのリリースは `.github/workflows/release.yml` により **完全自動化** されている。
`v*` タグをプッシュするだけで、GitHub Actions が全プラットフォームのビルドとリリース作成を行う。

**手動でのSEAバイナリビルドやリリース作成は不要。**

## リリース手順

```bash
# 1. mainブランチで変更をコミット・プッシュ
git push origin main

# 2. バージョンタグを打ってプッシュ
git tag v0.3.1
git push origin v0.3.1

# 3. 以上。CIが自動で全成果物をビルド・リリース作成する
```

タグプッシュ後、GitHub Actionsが約8〜10分で完了する。
進捗確認: `gh run list --limit 3`

## CIが自動ビルドする成果物一覧

| ファイル名 | ビルド環境 (CI) | 用途 |
|---|---|---|
| `ragmcp-server-darwin-arm64` | macos-latest | macOS用MCPサーバー / HTTPサーバー |
| `ragmcp-server-win32-x64.exe` | windows-latest | Windows用MCPサーバー / HTTPサーバー |
| `ragmcp-server-linux-x64` | ubuntu-latest | Linux用MCPサーバー / HTTPサーバー |
| `ragmcp-server-bundle.mjs` | ubuntu-latest | Node.jsバンドル版サーバー |
| `macos-release.zip` | macos-latest | macOS Flutterアプリ (SEAサーバー同梱) |
| `windows-release.zip` | windows-latest | Windows Flutterアプリ (SEAサーバー同梱) |
| `linux-release.tar.gz` | ubuntu-latest | Linux Flutterアプリ (SEAサーバー同梱) |
| `app-release.apk` | ubuntu-latest | Android Flutterアプリ (ローカルSQLite、サーバー不要) |

## CIワークフロー構成 (`.github/workflows/release.yml`)

1. **server_executables** — 3 OS (ubuntu/windows/macos) でSEAバイナリをビルド
2. **server_bundle** — esbuild単一ファイルバンドル
3. **flutter_macos** — macOS Flutterアプリ + SEAサーバー同梱
4. **flutter_linux** — Linux Flutterアプリ + SEAサーバー同梱
5. **flutter_windows** — Windows Flutterアプリ + SEAサーバー同梱
6. **flutter_android** — Android APK (サーバー不要、ローカルRAG)
7. **publish_release** — 全成果物をGitHubリリースにアップロード

## 使い方（エンドユーザー向け）

### MCPサーバーとして（Claude Desktop等）

```json
{
  "mcpServers": {
    "ragmcp": {
      "command": "/path/to/ragmcp-server-darwin-arm64",
      "args": ["--stdio"]
    }
  }
}
```

### HTTPサーバーとして（Flutter アプリ連携）

```bash
./ragmcp-server-darwin-arm64
# → http://127.0.0.1:3001 で起動
```

## 注意事項

- ローカルでの `npm run sea:build` は開発・デバッグ時のみ使用。リリースにはCIを使う
- Flutterアプリ (Android) はサーバー不要。ローカルSQLiteで動作する
- `gh release delete` でリリースを削除してもタグは残るので注意
