# リリース手順マニュアル

ragmcpのリリースはプラットフォームごとにSEAバイナリをビルドし、同一のGitHubリリースにアップロードする。SEAはクロスコンパイル不可のため、各OS上でビルドする必要がある。

## 前提

- Node.js >= 24
- npm依存パッケージがインストール済み (`npm install`)
- GitHubのpush権限と`gh` CLIがログイン済み

## 1. コード準備（どのマシンでも可）

```bash
# 1. mainブランチで変更をコミット・プッシュ
git add -A && git commit -m "fix: ..."
git push origin main

# 2. バージョンタグを打つ（例: v0.3.1）
git tag v0.3.1
git push origin v0.3.1
```

## 2. GitHubリリース作成（最初のマシンで実行）

```bash
# ビルド
npm run build
npm run sea:build

# リリース作成 + バイナリアップロード
# macOSの場合:
gh release create v0.3.1 \
  build/sea/ragmcp-server-darwin-arm64 \
  --title "v0.3.1" \
  --notes "変更内容をここに書く"

# Windowsの場合:
gh release create v0.3.1 \
  build/sea/ragmcp-server-win32-x64.exe \
  --title "v0.3.1" \
  --notes "変更内容をここに書く"
```

## 3. 別プラットフォームのバイナリを追加（2台目以降のマシンで実行）

```bash
# 最新コードを取得
git pull origin main

# ビルド
npm install
npm run build
npm run sea:build

# 既存リリースにバイナリを追加アップロード
# Windowsマシンの場合:
gh release upload v0.3.1 build/sea/ragmcp-server-win32-x64.exe

# macOSマシンの場合:
gh release upload v0.3.1 build/sea/ragmcp-server-darwin-arm64
```

## 成果物一覧

| ファイル名 | ビルド環境 | 用途 |
|---|---|---|
| `ragmcp-server-darwin-arm64` | macOS (Apple Silicon) | macOS用MCPサーバー / HTTPサーバー |
| `ragmcp-server-win32-x64.exe` | Windows (x64) | Windows用MCPサーバー / HTTPサーバー |

## 使い方（エンドユーザー向け）

### MCPサーバーとして（Claude Desktop等）

```json
{
  "mcpServers": {
    "ragmcp": {
      "command": "/path/to/ragmcp-server-darwin-arm64",
      "args": ["--stdio"]
    }
  }
}
```

### HTTPサーバーとして（Flutter アプリ連携）

```bash
./ragmcp-server-darwin-arm64
# → http://127.0.0.1:3001 で起動
```

## 注意事項

- SEAバイナリはクロスコンパイル不可。Windows用は必ずWindowsでビルドする
- `gh release upload` で既存リリースにバイナリを追加できる
- Flutterアプリ (Android) はサーバー不要。ローカルSQLiteで動作する
