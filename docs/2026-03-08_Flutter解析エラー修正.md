## 問題の概要

CIの `flutter analyze` ステップで354件のエラーが発生し、ビルドが失敗していた。

## 根本原因

コードのマージ時に、`local_rag_service.dart` で2つの異なる実装が不完全にマージされ、以下の3つの問題が混入していた。

## 修正内容

### 1. 孤立したSQLコードブロックの除去 (`local_rag_service.dart`)

**問題**: `_fullWidthToHalfWidth()` メソッドの直後に、文字列リテラル (`'''`) で囲まれていない生のSQLコードが存在していた。これが354件のパースエラーの主原因。

```dart
// 修正前: メソッド終了直後にSQLが露出
  }
            WHERE chunks_trigram MATCH ? AND source_id = ?
            ORDER BY rank ASC
            LIMIT ?
          ''', [escaped, sourceId, effectiveLimit]);
        } catch (_) {
          rows = [];
        }
      }
    }
    // ... 以下、2つ目の検索実装が続く
```

古い実装と新しい実装が混在し、古い方の開始部分（`db.select('''` と SQLのSELECT句）が削除されたが、WHERE句以降と後続のTier 3検索・結果マッピングが残っていた。新しい実装（行272-361）が正しい完全な実装なので、孤立した旧コード（約37行）を削除した。

### 2. 存在しない `Row` メソッドの修正 (`local_rag_service.dart`)

**問題**: `searchSource` の結果マッピングで `row.readText()` / `row.readReal()` を使用していたが、sqlite3パッケージ(v2.9.4)の `Row` クラスにはこれらのメソッドは存在しない。

```dart
// 修正前: 存在しないメソッド
return rows
    .map((row) => RagSearchResult(
      chunkId: row.readText('chunk_id'),
      score: row.readReal('rank'),
    ))
    .toList();

// 修正後: operator[] によるカラムアクセス + スコア計算
return rows.map((row) {
  final rank = (row['rank'] as num?)?.toDouble() ?? 1.0;
  final score = max(0, (100 - rank * 10).round());
  final content = (row['content'] ?? '') as String;
  return RagSearchResult(
    chunkId: (row['chunk_id'] ?? '') as String,
    documentId: (row['document_id'] ?? '') as String,
    relPath: (row['rel_path'] ?? '') as String,
    heading: row['heading'] as String?,
    content: content,
    excerpt: buildExcerpt(content, query),
    score: score,
  );
}).toList();
```

追加で `excerpt` フィールドも正しく生成するようにした（`buildExcerpt` は `search_helpers.dart` で定義済み）。

### 3. 存在しない `String.normalize()` の除去 (`local_rag_service.dart`)

**問題**: `_prepareSearchVariations()` 内で `query.normalize(NormalizationForm.nfkc)` を呼んでいたが、Dart標準ライブラリの `String` にはこのメソッドは存在しない（`characters` パッケージ等が必要）。

```dart

// 修正後: 該当行を削除
// ひらがな↔カタカナ変換、半角↔全角変換で十分カバーできる
```

### 4. `searchSource` の実装追加 (`rag_source_service.dart`)

**問題**: `RagServiceInterface` で定義されている `searchSource` メソッドが `RagSourceService`（HTTP版）に実装されておらず、空の `@override` アノテーションだけが残っていた。

```dart
// 追加した実装
@override
Future<List<RagSearchResult>> searchSource(
  String sourceId, String query, {int limit = 8}
) async {
  final response = await _client.post(
    _uri("/rag/sources/$sourceId/search"),
    headers: _jsonHeaders,
    body: json.encode({"query": query, "limit": limit}),
  ).timeout(
    const Duration(seconds: 30),
    onTimeout: () => throw HttpException("Connection timeout"),
  );
  _throwIfNeeded(response);

  final payload = json.decode(response.body);
  final rawList = payload["data"];
  if (rawList is! List) return [];
  return rawList
      .whereType<Map>()
      .map((item) => RagSearchResult.fromJson(
          Map<String, dynamic>.from(item)))
      .toList();
}
```

サーバー側のエンドポイント `POST /rag/sources/:sourceId/search` は `src/routes/rag.ts` に既に実装済み。
